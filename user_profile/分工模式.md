# 如何筛选用户：

task_pre:问一个真正的问题，问题组要有逻辑，问题要开放
task0: 关于主干问题
	不要问二面性问题
	在时间上的词语要点明，不要用模糊词语；
	主观性的词不要用；
	问题逻辑要清楚，第一个问题要有引子；
	不要让用户判断你的产品，让他间接阐述需求
	“最”---从吐槽角度开始，循序渐进地设计问题
	你的循序渐进问题不要诱导，主导他回答；
	不要带着自己的答案问问题；
	不要用自己的词语，句子要通俗简单，
